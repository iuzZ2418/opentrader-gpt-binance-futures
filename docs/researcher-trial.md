# 研究员试用与标注协议

## 目标

验证系统是否减少公告首读时间、能否发现研究员认为重要的增量信息，并用独立标注测量
事件抽取的误报和漏报。种子评测结果不得替代本试用。

## 建议样本

- 一个细分行业；
- 20至30家公司；
- 连续30天公开公告；
- 至少500个有效文本片段；
- 两名研究员独立重叠标注不少于100个片段；
- 剩余片段可由单人标注，再抽取10%复核。

抽样必须同时包含模型预测为事件和未预测为事件的片段，否则无法测量漏报。

## 标注步骤

1. 两名研究员使用不同标注人名称进入“片段标注”模式。
2. 不查看另一人的结果，选择 `event`、`no_event` 或 `uncertain`。
3. 若为事件，选择事件类型、方向和状态。
4. 对本体无法覆盖或原文歧义的片段写备注。
5. 每50条保存一次阶段结果并导出JSONL备份。
6. 完成重叠样本后调用一致性接口，逐条仲裁分歧。

一致性接口：

```text
GET /annotation/agreement?annotator_a=researcher-a&annotator_b=researcher-b
```

## 验收指标

- 事件/无事件 Cohen's kappa 建议不低于0.70；
- 事件类型精确率和召回率分别报告，不只报告准确率；
- 数字复制准确率不低于99%；
- 原文不存在的事实新增率不高于2%；
- 每日前20条事件中，研究员判定有价值的比例不低于60%；
- 记录单份公告首读耗时，与原工作流做配对比较。

## 反馈分类

- 抽取错误：事件本身不存在；
- 类型错误：事件存在但本体映射错误；
- 方向或状态错误；
- 证据不完整：遗漏限定条件、数字或因果边界；
- 漏检：人工标记为事件但模型没有输出；
- 排序错误：事件正确但研究价值排名明显不合理；
- 本体缺口：现有事件类型无法表达。

## 试用完成条件

形成带版本号的标注集、标注一致性报告、错误清单和下一轮规则或模型修改记录。任何模型
调整都应在冻结测试集上复评，避免只针对当前样本修补。
